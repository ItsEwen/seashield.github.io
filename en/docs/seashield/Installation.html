Installation of SeaShield

Where can I find SeaShield?

Réponse :

The answer is quite simple, join us on Discord and go to the store to subscribe

Once purchased, a license key will be assigned to you via a private message from the SeaShield Discord bot and the zip file containing SeaShield will be available on your keymaster mentioned during your purchase.

1 - Implement SeaShield on your FiveM server​

Go to your resources folder and drag the unzipped Seashield folder

You should have a folder like this in your resources folder

resources/SeaShield
    ├── seashield_banlist.sql  # SQL to implement in your DB
    ├── fxmanifest.lua  # Manifest to launch the SeaShield script
    ├── .fxap  # Protection Asset Escrow
    │            
    ├── client
    │   └── Several files protected by Asset Escrow
    │
    ├── configs # Configuration File
    │   ├── cl_config.lua
    │   ├── sh_utils.lua
    │   └── sv_config.lua
    │
    ├── server 
    │    └── Several files protected by Asset Escrow
    ├── stream 
    │    └── Several files protected by Asset Escrow
    └── vendor   
        └── Several files protected by Asset Escrow

2 - Configure SeaShield​

2.1 Configuration cl_config.lua​

AntiKillEngine​

I don't know | false by default

AntiBlips​

Bans players with blips on the map | false by default. (not recommended if you have blips. Example: Job Police with colleague visible on the map)

AntiFastRun​

Banned players in Super Speed | false by default

AntiFreecam​

Banned Freecam players | false by default (Ultra Sensitive, ByPass possibility available with exports. Expert Chapter)

AntiSpectate​

Bans players going through spectator mode to see other players | true by default on both options

Active = true,
BanActive = true,

AntiNoClip​

Banned players passing through the NoClip | false mode by default

Active = false,
PositionBypass = {
    vector3(0,0,0),     # Addition of the various positions not taken into account by SeaShield
},
BanActive = true,

AntiSound​

Prevents random, native or non-native sound from being played on all clients

AntiTextureStream​

Banned players using a forbidden Texture stream (Ideal for injected menus) | false by default

Active = false,
BanActive = true,
TextureName = {
    'commonmenu', --    #Example, add as much texture as you like
}

AntiThermalVision​

Banned players using heat vision | true by default

AntiNightVision​

Banned players using night vision | true by default

ESX​

Banned players using different Trigger Object than mentioned below | true by default
(Only for owners using ESX Framework)

Active = true,
Trigger = 'esx:getSharedObject'     #To be modified by your ESX Object trigger

AntiFlyVehicle​

Banned players who want to fly vehicles | true by default

MaxVehicleFly = 15  #By default

2.2 Configuration sh_utils.lua​

With the new version of SeaShield, sh_utils.lua allows the addition of your own logo and the translation of the different detections

2.3 Configuration sv_config.lua​

Lang​

Allows you to change the language of SeaShield | FR by default

ServerName​

Allows you to change the name of your Server | ServerName by default

License​

Allows you to change the name of your server to the default license.
This is where your license should be implemented!

devMode​

Enable Dev mode | false by default
(Enable logs to debug some functions)

UseStaffESX​

Enable Dev mode | false by default 

Active = true,
GroupList = {
    ['yourGroupESX'] = true,    #On ESX, the admin groups are: "admin" or "superadmin"
},
DefaultGroupName = 'user',  #In general, on ESX, the default user group is 'user'.

customBanSystem​

Allows you to use your own ban system instead of the default SeaShield system.

active = false, -- Set to true if you want to use the SeaShield system
banEvent = function(src, reason) -- If you set false, remember to add your ban trigger like the example below
    -- EXAMPLE: 
    TriggerEvent('player:anticheat:BanPlayer', src, reason)
end

deferralsCards​

Enable the visibility of the connection check when a player connects | true by default

allowStaff​

Do not use if "UseStaffESX" is active | true by default

['LicenseWL'] = true,

#EXEMPLE

['licence:15z4dqz68498498q4z6d8484zd8'] = true,

Webhooks​

This is where you will add the link of your Webhook
Here is a link to generate your webhooks if you can't do it

AntiCrash​

Blocks injectors that crash your server and blocks the /me flaw.

meCrash = false, -- true if you want the /me flaw to be patched. Don't forget to change the trigger like the example below
meCommand = function(src,args, rawCommand)
    TriggerClientEvent('::{IlIIIlIIllIIlIlI}::3dme:trigger', -1, src, 'Player '..rawCommand:sub(4))
end

AntiPlayerModel​

Prevents a player from using a non-whitelist PED model

BlacklistCrash​

Blocks crashes created by injectors or badly injected modmenu | (Not recommended)

ESX (Server Side)​

Configuration of ESX on the server side, only if you use the ESX framework.

UseESX = true,      # true Only if you use ESX
Trigger = 'esx:getSharedObject',    # ESX Trigger Object used on the server side (default : esx:getSharedObject)
BanCommand_Group = {
    ['fondateur'] = true        # Group ESX to allow the use of the ban command
},
UnbanCommand_Group = {
    ['fondateur'] = true        # Group ESX to allow the use of the deban command
},

VehicleMaxSpeed​

Bans a player if the vehicle speed exceeds the default value | true by default

MaxSpeed = 530

AntiGodMod​

Ban a player if a GodMod is activated | true by default

AntiInvisible​

Ban a player if he is invisible | true by default

AdminMenu​

Allows access to the Admin Menu directly in game | true by default

BypassESX = true,
BypassList = {
    ['Group'] = true, -- If you use ESX, indicate your different groups 
    ['license:...'] = true, -- If you are not using ESX, specify the different licenses allowed
},

AntiStopEulen​

Ban a player trying to stop a resource | true by default

list = {
    SeaShield = true, -- If you want to add resources protected by anti-stop, add them here.
}

AntiDesudo​

Ban a player using the Desudo injector | true by default

MassDeleteVehicle​

Ban a player who is definitely using a massive vehicle removal | true by default

MaxVehicleDelete = 5, -- Conseillé à 15

AntiWeaponDamageModifier​

Ban a player who changes the damage of a weapon | true by default

MaxDamageMeleeMultiplier = 5.0,     # Recommended : 5.0
MaxDamageWeaponMultiplier = 5.0,    # Recommended : 5.0

UseScreenShot​

Enables the display of a screenshot of the player screen at the time of his ban in the webhooks | true by default

Vehicles​

Allows banning a player trying to spawn a blacklisted vehicle | true by default
(You can add or remove as many vehicles as you like. The current list is blacklisted on most RP servers)

Explosions​

Ban a player who has spawned multiple explosions in a short period of time | true by default
(The default values are largely necessary, if you encounter false positives then increase the following options)

LimitMassExplosionEvent = 25

Peds​

Allow spawn of PEDs specified in the whitelist | true by default

#TO ADD A PED
[GetHashKey('a_f_o_soucent_01')] = true,

#TO ADD A PED WITH THE HASH
[-1354005816] = true,

Props​

Allow spawn of Props specified in whitelist | true by default

#TO ADD A PROPS
[GetHashKey("prop_pencil_01")] = true,

#TO ADD A PROPS WITH THE HASH
[-1354005816] = true,

#Option 

LimitMassSpawnProps = 40,   -- Increase if necessary
AntiMassPropsDelay = 10, -- Secondes

Particles​

Allow the spawn of the particles indicated in the whitelist | true by default

#TO ADD A PARTICLE
[GetHashKey('water_splash_plane_trail')] = true,

#TO ADD A PARTICLE WITH THE HASH
[-1354005816] = true,

FireEvent​

I don't know | true by default

AntiProjectileEvent​

Bans players using projectiles | true by default

AntiGiveWeapon​

Bans players who want to give themselves a weapon | true by default

AntiRemoveWeapon​

Bans players who want to remove weapons | true by default

AntiRemoveAllWeapon​

Bans players who want to remove all weapons by default

AntiClearPedTasks​

Bans players who want to run a ClearPedTask | true by default

AntiGiveWeaponESX​

Bans players who want to give themselves a weapon and use the ESX framework | true by default

AntiShoot​

Bans players who want to shoot a weapon | true by default

BlacklistWeapons​

Ban players who want to spawn a blacklisted weapon | true by default
(The list is more than enough, if you want to add or remove a weapon from the list, there is no particular problem)

#TO ADD A WEAPON
[GetHashKey('weapon_marksmanpistol')] = true,

AntiSpamTriggers​

Bans players trying to spam a particular trigger. | true by default | TriggersList allows you to list triggers to protect 

#TO ADD A TRIGGER

TriggersList = {
    'trigger1',
    'trigger2',
    'trigger3',
},

#TO ADJUST THE RATE LIMITER

NumberLimit = 15,   #IT'S RIGHT HERE

BlacklistEvents​

Bans players trying to use a Blacklist trigger | true by default | TriggersList allows to list triggers to protect 

#TO ADD A TRIGGER

TriggersList = {
    'trigger1',
    'trigger2',
    'trigger3',
},

ListWeapon​

Avoid touching this list which allows the anti give weapon
