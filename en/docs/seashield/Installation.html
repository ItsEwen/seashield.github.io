2 - Configure SeaShield‚Äã

2.1 Configuration cl_config.lua‚Äã

AntiKillEngine‚Äã

I don't know | false by default

AntiBlips‚Äã

Bans players with blips on the map | false by default. (not recommended if you have blips. Example: Job Police with colleague visible on the map)

AntiFastRun‚Äã

Banned players in Super Speed | false by default

AntiFreecam‚Äã

Banned Freecam players | false by default (Ultra Sensitive, ByPass possibility available with exports. Expert Chapter)

AntiSpectate‚Äã

Bans players going through spectator mode to see other players | true by default on both options

Active = true,
BanActive = true,

AntiNoClip‚Äã

Banned players passing through the NoClip | false mode by default

Active = false,
PositionBypass = {
    vector3(0,0,0),     # Addition of the various positions not taken into account by SeaShield
},
BanActive = true,

AntiSound‚Äã

Prevents random, native or non-native sound from being played on all clients

AntiTextureStream‚Äã

Banned players using a forbidden Texture stream (Ideal for injected menus) | false by default

Active = false,
BanActive = true,
TextureName = {
    'commonmenu', --    #Example, add as much texture as you like
}

AntiThermalVision‚Äã

Banned players using heat vision | true by default

AntiNightVision‚Äã

Banned players using night vision | true by default

ESX‚Äã

Banned players using different Trigger Object than mentioned below | true by default
(Only for owners using ESX Framework)

Active = true,
Trigger = 'esx:getSharedObject'     #To be modified by your ESX Object trigger

AntiFlyVehicle‚Äã

Banned players who want to fly vehicles | true by default

MaxVehicleFly = 15  #By default

2.2 Configuration sh_utils.lua‚Äã

With the new version of SeaShield, sh_utils.lua allows the addition of your own logo and the translation of the different detections

2.3 Configuration sv_config.lua‚Äã

Lang‚Äã

Allows you to change the language of SeaShield | FR by default

ServerName‚Äã

Allows you to change the name of your Server | ServerName by default

License‚Äã

Allows you to change the name of your server to the default license.
This is where your license should be implemented!

devMode‚Äã

Enable Dev mode | false by default
(Enable logs to debug some functions)

UseStaffESX‚Äã

Enable Dev mode | false by default 

Active = true,
GroupList = {
    ['yourGroupESX'] = true,    #On ESX, the admin groups are: "admin" or "superadmin"
},
DefaultGroupName = 'user',  #In general, on ESX, the default user group is 'user'.

customBanSystem‚Äã

Allows you to use your own ban system instead of the default SeaShield system.

active = false, -- Set to true if you want to use the SeaShield system
banEvent = function(src, reason) -- If you set false, remember to add your ban trigger like the example below
    -- EXAMPLE: 
    TriggerEvent('player:anticheat:BanPlayer', src, reason)
end

deferralsCards‚Äã

Enable the visibility of the connection check when a player connects | true by default

allowStaff‚Äã

Do not use if "UseStaffESX" is active | true by default

['LicenseWL'] = true,

#EXEMPLE

['licence:15z4dqz68498498q4z6d8484zd8'] = true,

Webhooks‚Äã

This is where you will add the link of your Webhook
Here is a link to generate your webhooks if you can't do it

AntiCrash‚Äã

Blocks injectors that crash your server and blocks the /me flaw.

meCrash = false, -- true if you want the /me flaw to be patched. Don't forget to change the trigger like the example below
meCommand = function(src,args, rawCommand)
    TriggerClientEvent('::{IlIIIlIIllIIlIlI}::3dme:trigger', -1, src, 'Player '..rawCommand:sub(4))
end

AntiPlayerModel‚Äã

Prevents a player from using a non-whitelist PED model

BlacklistCrash‚Äã

Blocks crashes created by injectors or badly injected modmenu | (Not recommended)

ESX (Server Side)‚Äã

Configuration of ESX on the server side, only if you use the ESX framework.

UseESX = true,      # true Only if you use ESX
Trigger = 'esx:getSharedObject',    # ESX Trigger Object used on the server side (default : esx:getSharedObject)
BanCommand_Group = {
    ['fondateur'] = true        # Group ESX to allow the use of the ban command
},
UnbanCommand_Group = {
    ['fondateur'] = true        # Group ESX to allow the use of the deban command
},

VehicleMaxSpeed‚Äã

Bans a player if the vehicle speed exceeds the default value | true by default

MaxSpeed = 530

AntiGodMod‚Äã

Ban a player if a GodMod is activated | true by default

AntiInvisible‚Äã

Ban a player if he is invisible | true by default

AdminMenu‚Äã

Allows access to the Admin Menu directly in game | true by default

BypassESX = true,
BypassList = {

list = {
    SeaShield = true, -- If you want to add resources protected by anti-stop, add them here.
}

AntiDesudo‚Äã

Ban a player using the Desudo injector | true by default

MassDeleteVehicle‚Äã

UseScreenShot‚Äã

Enables the display of a screenshot of the player screen at the time of his ban in the webhooks | true by default
